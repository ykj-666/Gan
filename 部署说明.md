# 🚀 Gan 项目部署说明（纯小白版）

## 一、准备工作（只做一次）

### 1. 安装 Node.js

如果您的电脑还没有安装 Node.js：

1. 打开浏览器访问：https://nodejs.org/dist/v20.18.0/node-v20.18.0-x64.msi
2. 下载后双击安装，全部点击"下一步"即可
3. 安装完成后重启电脑

**注意**：必须安装 **v20 或更高版本**。旧版本会导致部署失败。

### 2. 确认项目代码在本地

确保您已经把代码下载/克隆到本地文件夹，比如 `D:\Gan`

---

## 二、一键部署到服务器

### 方法一：双击运行（推荐）

1. 打开项目文件夹
2. 优先双击 **`打开部署网页.bat`**
3. 会自动打开浏览器，显示图形化部署界面
4. 如果您更习惯命令行，也可以双击 **`点击部署.bat`** 打开命令行菜单
5. 即使发生错误，命令行窗口现在也会保留，不会一闪而过

### 方法二：命令行运行（命令行菜单）

1. 在项目文件夹的地址栏输入 `cmd`，按回车
2. 输入命令：`node deploy.cjs`
3. 回车运行

### 方法三：Web 界面

1. 在项目文件夹的地址栏输入 `cmd`，按回车
2. 输入命令：`node deploy-server.cjs`
3. 会自动打开浏览器，显示图形化部署界面

## 三、操作说明

运行后会看到菜单：

```
╔══════════════════════════════════════════╗
║       Gan 项目一键部署工具               ║
║       服务器: 118.89.134.207            ║
╚══════════════════════════════════════════╝

  [1] 一键部署到服务器（首次或更新）
  [2] 启动本地开发服务器（本地测试）
  [3] 查看服务器运行状态
  [4] 查看部署说明
  [5] 退出
```

### 首次部署（选择 1）

1. 按 `1`，回车
2. 如果是第一次，会提示输入 **服务器 ubuntu 密码**
3. 输入密码（输入时不会显示，这是正常的），回车
4. 等待自动完成（大约 2-5 分钟）
5. 看到绿色 "部署成功" 提示即可

### 更新代码（选择 1）

代码修改后：
1. 双击 `打开部署网页.bat` 或 `点击部署.bat`
2. 选择部署
3. **不需要再输入密码**，自动完成更新

### 本地测试（选择 2）

不想部署到服务器，只想在本地测试：
1. 按 `2`，回车
2. 自动启动本地服务器
3. 浏览器打开 http://localhost:3000
4. 关闭黑色窗口即可停止

## 五、常见问题

### ❓ 部署后打不开网页

**原因：** 腾讯云安全组没有开放 3000 端口

**解决方法：**
1. 打开腾讯云控制台
2. 找到您的服务器 → 安全组
3. 点击「入站规则」→「添加规则」
4. 填写：协议 TCP，端口 3000，来源 0.0.0.0/0
5. 保存，稍等几分钟再试

### ❓ 提示 "未检测到 Node.js" 或 "Node.js 版本过低"

**原因：** 电脑没有安装 Node.js，或版本低于 v20

**解决方法：** 看上面的「一、准备工作」，安装 v20 以上版本

### ❓ 提示 "SSH 连接失败"

**原因：** 服务器 22 端口未开放，或密码错误

**解决方法：**
1. 确认服务器 IP 是 `118.89.134.207`
2. 确认 ubuntu 密码正确
3. 在腾讯云安全组中确认开放了 TCP 22 端口

### ❓ 提示 "检测到未提交的更改"

**原因：** Git 工作区有未提交的修改

**解决方法：**
- 建议先执行 `git add . && git commit -m "更新"` 再部署
- 或者输入 `y` 强制继续部署

### ❓ 更新后数据会丢失吗？

**不会。** 数据库是 SQLite 文件（`local.db`），每次更新时会自动备份和恢复。
